## Allowed scope

- Only use helpers listed in `helper_manual.md`.

## Object return rules

- Single-part assets may return one mesh object directly.
- Multi-part assets must either:

  - parent all parts under one root object and return the root.

## Determinism

- All randomness must be controlled by `factory_seed`.

## Scene safety

- Avoid depending on current scene contents.
- Avoid selecting unrelated objects.
- Avoid absolute paths.
- Avoid file I/O.
- Avoid assumptions about existing collections, materials, or assets beyond the local factory code.

## Geometry style

- Prefer simple primitives plus transforms.
- Prefer a small number of modifiers: `SUBSURF`, `BEVEL`, `SOLIDIFY`, `DISPLACE`, or one simple geometry-nodes deformation.
- Prefer a small number of materials with simple shader graphs.
- Keep topology and node graphs modest.

- Use `surface.add_material` with a small shader function.

- Avoid deep node-group nesting.

- Set `location`, `rotation_euler`, and `scale` explicitly.
- Apply transforms when later modifiers or joining depend on final dimensions.
- Name the final returned object.

## Error avoidance

- Do not return Python lists.
- Do not return `None`.
- Do not leave loose temporary helper objects in the scene unless they are joined or parented into the asset.
